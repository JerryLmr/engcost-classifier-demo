# PROJECT STATE

## Current Milestone
- Demo 已实现“输入物业工程名称 -> 输出一级/二级工程类别”的单条分类与 Excel 批量分类。

## Completed Work
- 搭建了 FastAPI 后端接口：`/api/health`、`/api/classify`、`/api/classify-excel`。
- 搭建了前端静态页面，支持单条分类与 Excel 上传下载。
- 分类策略已调整为“先判一级，再在一级内判二级”。
- 下调了泛化关键词权重，例如“管道”“交通”，减少误判。
- 为 LLM 调用失败和无效返回增加了可控降级结果，避免接口直接返回 500。
- 后端已按职责拆分为 `api / core / data / models / services` 模块。
- 前端已拆分为独立的 `HTML / CSS / JS` 文件，并支持集中配置 `API_BASE`。
- 已补充基础自动化测试，覆盖规则分类、降级兜底和 API 基本可用性。
- 已为高冲突类别增加“边界口径配置层”，支持先按业务边界判定再做分类打分。
- 已将防水/外立面、消防、电梯、监控的关键二级分类改为“对象词 + 动作词”判定。
- 已补充边界回归测试，覆盖外立面漏水、防火门更新、监控改造、电梯部件更换等真实冲突样本。

## Decisions
- 当前阶段优先保证演示稳定性和可解释性，不引入训练集、数据库或复杂检索。
- 保留“规则优先 + LLM 兜底”的策略，用于平衡稳定性与覆盖率。
- 降级默认分类暂定为“公共设施 / 公共区域维修”，后续可按业务反馈调整。

## Known Limitations
- 分类体系和规则仍以 Python 常量形式维护，尚未迁移到 JSON 或数据库。
- 当前规则未经过系统样本回归，准确率仍需真实工程名称样本验证。
- 前端 `API_BASE` 仍默认指向 `http://127.0.0.1:8000`，跨环境部署时需要显式配置。
- 当前“待人工确认/低置信度”仍未对外暴露，规则不确定时仍只能走 LLM 或默认降级。
- API 自动化测试在缺少 `httpx` 时会被跳过，测试依赖仍需补齐。

## Manual User Edits
- 无记录。

- 将分类配置和规则迁移到 JSON 或可编辑配置文件，降低维护成本。
